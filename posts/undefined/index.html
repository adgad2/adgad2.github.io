使用JS使仿制页实现真正的百度搜索功能V2

发表于 2017-05-19   |   分类于 技术向

5.20再次更新：修复在搜索框按回车键后，无论confirm返回值为1或0，都会导致页面刷新再次弹出alert的问题；修复搜索框在不同浏览器窗口大小下不居中的问题。

继html+css后稍微学了下JavaScript，于是想利用JavaScript的功能让仿制的百度页面可以真正地进行搜索。

修改后的仿制网页：点击打开

首先写好JavaScript的代码部分，放在head里，代码：

<script type="text/javascript">
    alert("该网页仅做个人学习使用，不用做任何商业用途");
    function baidu(){//打开百度搜索页
        var key=document.getElementById('search').value;
        r=confirm("注意！以下将会跳出本博客打开百度搜索结果页面，请按\“确定\"继续");
        if(r){
            window.open("https://www.baidu.com/s?ie=UTF-8&wd="+key,"_blank")
        }//这里第一次出错了，记住要加上https://使打开绝对路径
    }
    function EnterPress(){//响应文本框的回车
        if(event.keyCode == 13){
            baidu();
        }
    }

</script>

然后给form里的搜索框和按钮加上响应事件：

<form >
    <span><input id='search' type="text" name="search" onkeypress="EnterPress()"></span><span><input id="button" type="button" value="百度一下" onclick="baidu()"></span>
</form>

OK，大功告成。

V2更新：测试之后发现一问题，在搜索框按下回车后会导致页面刷新的问题。

一开始以为是事件keypress的原因，找了很久，发现单击“百度一下”按钮并不会触发该问题，Google许久后发现是form表单的问题:
如果一个form标签中只有一个文本框 <input type="text" />的话，在框中输入数据回车后，就会触发该问题。
解决方法：在form标签中再增加一个隐藏的text文本框，代码如下：

<input type="text"  style="visibility:hidden;" />

不居中的解决方法：给form加一个宽度，然后设置margin:auto;
